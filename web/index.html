· MA2 Timecode Tools

web tool · also on the desktop

MA2 Timecode Tools

Cut a section from a track and the timecode show has to collapse with it — drop the cues inside the removed window, slide everything after it left by the exact length of the cut. MA2 Timecode Tools runs that ripple across every subtrack at once and hands back a file grandMA2 re-imports without a fight.

Open the tool ▸ Desktop build ↓ Source

Cut · insert · erase · move

Ripple-cut a window, open a gap, erase in place, or drag a single cue — each snapping to the bar grid.

Byte-exact

BOM, namespace, tabs, line endings — everything but the edit stays untouched. The import never chokes.

Bars, not seconds

Work in whole bars at the track BPM, or frame-accurate by timecode. A live preview names every cue that moves.

Stays on your machine

Everything runs in the browser. Nothing is uploaded — no server sees your show.

Export from grandMA2

The tool operates on the XML grandMA2 exports from a Timecode pool object. Command line — one line, unambiguous:

Export Timecode 14 "tc_show"
  • 14 — the number of the Timecode pool object to export.
  • "tc_show" — output filename, without extension (MA appends .xml).

It writes into gma2/importexport on the selected drive. Copy tc_show.xml to the machine running this tool.

Import back into grandMA2

Import puts the filename first (the “NAME”), destination after; the .xml extension is not written:

Import "SHOW_cut" At Timecode 40

Import Timecode 40 "…" gives Error #12: EXPECTED NAME — the order is the reverse of Export.

Import into an empty slot, keep the original as a backup, and check the length.

· LTC Generator

web tool

LTC Generator

Render SMPTE Linear Timecode (LTC) WAV files in the browser. A single file with a custom start TC, length and FPS — or a whole day's batch in one go: any range, any interval, any length per file, downloaded as a ZIP. Drop-frame supported for 29.97.

Open the tool ▸

SMPTE-correct

80-bit packet with parity, biphase-mark-coded at 80×fps, decoded by every console and DAW that reads LTC.

Batch a whole day

"00:00 → 23:59, every 30 min, 25 min each" gets you 48 files in one ZIP — no spreadsheets.

Any rate

24, 25, 29.97 (DF or NDF), 30. Sample rate 22.05 / 44.1 / 48 kHz, level in dBFS.

Stays on your machine

Everything renders in the browser. Nothing is uploaded.

· ØJE Cue Monitor

desktop app · macOS & Windows

ØJE Cue Monitor

Timecode comes in over the PA and the cues have to fire on the hit, in the order you set — not sorted. ØJE Cue Monitor reads LTC/SMPTE off any audio input, triggers each cue by the frame it's pinned to, and throws a full-screen Performance view to the operators — lighting, audio, stage manager — each reading only their own column. The same view goes out over the local network to any phone.

Download ↓ macOS · Windows Source

Fires on the hit

Cues trigger by timecode in the order you set, not sorted — non-linear shows behave.

Every operator, one column

Role-coloured columns — lighting blue · audio amber · stage manager purple — so each operator finds their cues at a glance.

A remote on every phone

The Performance view goes out over the local network — password-protected, filtered so each phone shows only its operator's cues.

Knows the signal

Reads LTC/SMPTE off any input, with a five-bar VU and live · weak · clipping · no-signal states.
